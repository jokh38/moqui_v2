Detailed Description

A simple command-line interface parser for tests and applications.

This class provides a basic framework for parsing command-line arguments. It predefines a set of expected options (e.g., "--dicom_path") and stores any provided values for them. For a Python developer, this is similar in purpose to Python's argparse module, but much simpler.

Its main purposes are:

  1. Reading input files (e.g., RT-Ion plans).
  2. Configuring geometries, patients, dose grids, and beamline components.
  3. Setting up beam sources and simulation parameters (e.g., number of histories).

const std::vector< std::string > mqi::cli::operator[] ( const std::string & t)
inline

Provides read-only access to the arguments for a specific option.

This "overloaded operator" allows you to use square brackets [] to access the parsed arguments in a clean, dictionary-like syntax. Example: std::vector<std::string> energies = my_cli["--source_energy"];

Parameters
[in]tThe name of the command-line option (e.g., "--dicom_path").
Returns
A const vector of strings containing the arguments for the specified option. Returns an empty vector if the option was not provided on the command line.

void mqi::cli::read ( int argc,
char ** argv )
inline

Parses the command-line arguments and populates the parameters map.

This function iterates through the command-line arguments (argv). When it encounters a known option (a key that exists in the parameters map), it consumes all subsequent arguments as values for that option, stopping when it finds the next option (a string starting with "--") or runs out of arguments.

Parameters
[in]argcThe argument count, automatically supplied to main.
[in]argvThe argument vector (an array of C-style strings), automatically supplied to main.

◆ parameters

std::map<const std::string, std::vector<std::string> > mqi::cli::parameters
protected

A map to store the command-line options and their arguments. The key is the option name (e.g., "--dicom_path"), and the value is a vector of string arguments provided for that option. For a Python developer, this is analogous to a dictionary where keys are strings and values are lists of strings, e.g., ‘{’–nhistory': ['10000']}`.
