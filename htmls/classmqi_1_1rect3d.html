Detailed Description

template<typename T, typename R>
class mqi::rect3d< T, R >

A template class for a 3D rectilinear grid, supporting non-uniform spacing.

This class is a fundamental data structure for representing any data that exists on a 3D grid where the spacing between points on an axis is not necessarily uniform. It is designed to store data such as CT Hounsfield Units, calculated dose values, or deformation vector fields (DVFs).

It provides methods for construction, data access, trilinear interpolation, and various geometric calculations. The class uses raw pointers for its coordinate arrays (x_, y_, z_) and manages their memory manually with new[] in the constructors and delete[] in the destructor.

For a Python developer, this is analogous to having three 1D NumPy arrays for the coordinates and a 3D NumPy array for the data, but with C++'s strong typing and manual memory management.

Template Parameters
TThe data type of the grid values (e.g., int16_t for CT, float for dose, mqi::vec3<float> for a DVF).
RThe data type for the grid coordinates (e.g., float or double for precision).

◆ cell_corner

template<typename T, typename R>
enum mqi::rect3d::cell_corner

Enumerates the 8 corners of a grid cell (voxel) for interpolation.

The naming convention cXYZ indicates the corner's relative position, where 0 is the lower bound and 1 is the upper bound on each axis. For example, c000 is the corner with the minimum (x,y,z) coordinates in the cell, and c111 is the corner with the maximum.

A friend function to interpolate new rect3d from a source interpolate(ct, dose) : possible interpolate(dose, dose) interpolate(dvf, dose) : is not possible

A friend function to warp source data (rect3d) to destination (rect3d) using dvf. It will pull src data from ref thus DVF of ref->src is neccessary. In MIM, when we calculate DIR, ref should be chosen first. It is recommended that src, dest, dvf have all same resolution.

This function implements a "pull" or "backward" warping. For each voxel in the destination grid, it uses the DVF to find the corresponding point in the source grid's space and then interpolates the source data at that new point.

Template Parameters
T0Data type of the source and destination grids.
R0Coordinate type of all grids.
S0Scalar type of the DVF vectors.
Parameters
srcThe source grid containing the data to be warped.
destThe destination grid to be filled with the warped data.
dvfThe deformation vector field, which maps points from the destination space to the source space.
fill_valueThe value to use if the warped point falls outside the source grid.

< Number of voxels: Nx, Ny, Nz

< Looping reference

If destination point is in DVF grid points, apply translation and then check new position is in source Then, assign value by interpolating values at 8 coners surrounding new position.
