Detailed Description

template<typename T>
class mqi::treatment_session< T >

Manages a radiotherapy treatment session, acting as the primary interface to the Monte Carlo engine.

This class is the main entry point for a simulation. It reads and parses RT-ION DICOM files (Plan or Treatment Record), determines the modality, and instantiates the appropriate treatment machine model. It provides methods to access various components of the treatment plan, such as the beamline, beam source, and coordinate systems for each beam.

Template Parameters
TThe floating-point type for phase-space variables (e.g., float or double).
Note
Future development may include patient and dose grid management within this class.

Constructs a treatment session from a DICOM file.

This constructor reads the specified DICOM file, determines the modality (e.g., IONPLAN, IONRECORD), parses the dataset, and creates the corresponding treatment machine model.

Parameters
file_for_tx_machinePath to the DICOM file (RTPLAN, IONPLAN, RTRECORD, IONRECORD).
m_nameOptional machine name. If empty, it's read from the DICOM file.
mc_codeThe name and version of the Monte Carlo code being used (e.g., "TOPAS 3.5").
gantryNumThe gantry number, used to select the specific nozzle/machine model.
Note
Currently, only IONPLAN and IONRECORD modalities are fully supported.
